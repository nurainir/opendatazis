Linked Data

Seiring dengan keterbukaan data, mulailah para pengembang web dunia seperti Google dan Yahoo membuka akses data ke khalayak umum dengan memakai API sehingga dapat dimanfaatkan oleh pengembang web lain. Beda pengembang, tentunya berbeda pulalah cara pengaksesannya. Ketiadaan standar inilah yang membuat Tim Berner Lee (penemu WWW) memperkenalkan bagaimana cara mempublikasikan data dan menghubungkan antar data yang selanjutnya dikenal sebagai Linked Data. Linked Data adalah segala tentang penggunaan web untuk menampilkan, membagi dan menjalin data dan informasi dalam format RDF. Secara teknologi dapat dikatakan bahwa ide dasar dari Linked Data ini adalah penggunaan HTTP URI tidak hanya untuk mengidentifikasi halaman web namun juga memberikan identitas untuk entitas data. Cara ini memungkinkan kita memakai data yang berasal dari berbagai sumber. Pada prinsipnya, Linked Data harus memenuhi 4 syarat berikut :

  1. Menggunakan URI dalam pemberian nama apapun.
  2. Menggunakan HTTP URI agar semua orang bisa melihat nama tersebut.
  3. Ketika orang melihat sebuah URI maka akan tertampil informasi yang berguna dalam format RDF.
  4. Menambahkan  pernyataan  RDF  yang  mengubungkan  dengan  ke  URI  lain sehingga orang dapat menemukan hal yang terkait dengan data tersebut.

SPARQL

Berbagai data mulai dipublikasikan dengan mengadopsi prinsip Linked data. Data-data tersebut tidak disimpan di dalam database, namun disimpan dalam sebuah repositori RDF. Sebagian besar repositori tersebut menyediakan SPARQL Endpoint agar dapat diakses dengan mudah dengan perintah SPARQL yang mirip dengan perintah query pada database biasa.
